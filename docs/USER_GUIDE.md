# 📋 DesktopTodo — 桌面待辦清單 使用手冊

一款輕巧的桌面待辦事項管理工具，支援分類標籤、定時提醒，以及 Telegram 推播通知。  
適用於 Windows 與 macOS。

---

## 📥 安裝與啟動

### 方式一：直接執行 Python 腳本（需安裝 Python 3.8+）
1. 確認電腦已安裝 Python 3（[下載 Python](https://www.python.org/downloads/)）
2. 將 `DesktopTodo.py` 存放到任意資料夾
3. 雙擊 `DesktopTodo.py`，或在終端機輸入：
   ```
   python DesktopTodo.py
   ```

### 方式二：使用打包好的執行檔
- Windows：直接雙擊 `DesktopTodo.exe`
- macOS：直接雙擊 `DesktopTodo.app`

> 💡 本程式**不需要安裝任何額外套件**，使用 Python 內建功能即可運行。

## ✏️ 新增待辦事項

1. 在畫面下方的下拉選單中，選擇一個 TAG（例如 `#T`）
2. 在輸入框中打入待辦事項內容
3. **（選填）** 在右方 `HH:MM` 欄位輸入預計提醒時間（例如 `15:30`）
4. 按下 `Enter` 鍵或點擊「新增」按鈕

> 📌 如果您設定了預計時間，當系統時鐘到達該時間時，程式會**自動彈出提醒視窗**。

## 🏷️ TAG 標籤系統

TAG 標籤用來分類您的待辦事項，並決定它在「總結」時的行為。

### 預設 TAG

| TAG | 前置詞 | 推播 | 行為說明 |
|:---|:---|:---|:---|
| `#T` | `TODO-` | ✅ 是 | 總結時推播到 Telegram，推播後**自動從清單移除** |
| `#I` | `IDEAL-` | ✅ 是 | 總結時推播到 Telegram，推播後**自動從清單移除** |
| `#W` | `WORK-` | ❌ 否 | 不推播，留在清單中繼續追蹤 |
| `無` | — | ❌ 否 | 無標籤的一般待辦事項 |

### 自訂 TAG

您可以透過設定介面（⚙️）自由新增或刪除 TAG 規則：
- **名稱**：下拉選單中顯示的簡稱（例如 `#M`）
- **前置詞**：畫面與推播中使用的標籤文字（例如 `會議-`）
- **推播開關**：決定這類待辦在總結時是否要發送到 Telegram

## 📤 總結功能

### 自動總結
- 勾選「每日自動總結」後，程式會在設定的時間（預設 17:00）自動執行總結
- 您可以修改時間欄位來調整觸發時間

### 手動總結
- 點擊畫面下方的「執行手動總結」按鈕即可立即執行

### 總結時發生什麼？

```
總結觸發
 │
 ├─ ① 已完成項目 → 寫入 history_2026.md（歷史紀錄歸檔）→ 從清單移除
 │
 ├─ ② 未完成 + 推播=是 → 發送到 Telegram → 寫入 push_record_2026.md（備查）→ 從清單移除
 │
 └─ ③ 未完成 + 推播=否 → 繼續留在清單上
```

## ⚙️ 設定 Telegram 推播

點擊程式右上角的 `⚙️` 按鈕，開啟設定視窗。

### 步驟一：建立 Telegram Bot
1. 打開 Telegram，搜尋 **@BotFather**
2. 向它傳送 `/newbot`
3. 依照指示設定 Bot 名稱
4. BotFather 會回覆一段 **Token**（格式如 `123456789:ABCDEF-xxxxx`）
5. 將這段 Token **完整複製**，貼到設定視窗的「TG Token」欄位

### 步驟二：取得您的 Chat ID
1. 在 Telegram 中，**對您剛建立的 Bot 傳送任意訊息**（例如 `hi`）
2. 回到程式的設定視窗，點擊「🔍 自動取得 Chat ID」按鈕
3. 程式會自動取得並填入您的 Chat ID

### 步驟三：儲存
點擊「💾 儲存並套用設定」即可。

> ⚠️ **注意**：Chat ID 是一串純數字（例如 `950973637`），**不是** Bot 的名稱（如 @my_bot）。如果填錯格式，程式會自動提醒您。

## 📁 資料儲存位置

所有資料檔案統一存放在以下路徑的隱藏資料夾中：

| 作業系統 | 路徑 |
|:---|:---|
| Windows | `C:\Users\你的帳號\.desktoptodo\` |
| macOS | `/Users/你的帳號/.desktoptodo/` |

### 資料夾中的檔案

| 檔案 | 說明 |
|:---|:---|
| `todo_data.json` | 您的設定與待辦清單（程式唯一的資料來源） |
| `history_2026.md` | 歷史紀錄：所有已完成事項的年度歸檔 |
| `push_record_2026.md` | 推播備查：所有已推送到 Telegram 的項目紀錄 |

> 💡 歷史紀錄與推播備查檔案以年份自動分檔（例如 2026 年的紀錄存在 `history_2026.md`），不需手動管理。

## ❓ 常見問題

### Q: 推播失敗怎麼辦？
1. 確認網路連線正常
2. 點擊 ⚙️ 檢查 Token 與 Chat ID 是否正確填入
3. 確認 Token 是完整的（包含冒號和冒號後的字串）
4. 若 Chat ID 為空，請對您的 Bot 傳送一條訊息，然後使用「自動取得 Chat ID」功能

### Q: 我換了一台電腦，資料可以帶過去嗎？
可以。將 `~/.desktoptodo/` 資料夾整個複製到新電腦的相同位置即可。

### Q: 勾選完成後再取消勾選，會怎樣？
取消勾選後，項目會回到「未完成」狀態，且完成時間記錄會被移除。只要尚未執行總結，資料就不會被歸檔。

### Q: 關閉程式後資料會消失嗎？
不會。所有待辦事項都儲存在 `todo_data.json` 中，下次開啟程式時會自動載入。

### Q: 我可以同時在兩台電腦使用嗎？
可以，但兩台電腦的資料**不會自動同步**。每台電腦都有自己獨立的待辦清單。  
如果需要同步，可以將 `~/.desktoptodo/` 資料夾放到雲端同步服務（如 OneDrive、Google Drive）中。
